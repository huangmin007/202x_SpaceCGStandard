### 完成 RPCClientBase 完整实现
- 将占位类 `RPCClientBase` 重写为完整抽象基类，与 `RPCServerBase` 镜像对称设计
- 核心特性：TCP 连接管理、环形缓冲 CRLF 行拆分、请求/响应 Id 匹配、超时控制、可选自动重连
- 公共 API：`InvokeAsync<T>()`/`InvokeAsync()`（请求-响应）、`NotifyAsync()`（单向通知）
- 协议层抽象：`SerializeInvokeMessage()` / `ParseResponseMessage()` 两个抽象方法由子类实现
- 响应匹配：`ConcurrentDictionary<int, PendingCall>` + `TaskCompletionSource`
- 发送序列化：`SemaphoreSlim(1,1)` 防止并发写入字节交错
- 定义 `RPCResult` / `RPCResult<T>` 结果结构体
- 不使用 AutoReconnectTcpClient，TCP 逻辑全内置

### RPCClientBase 重构
- `RPCResult` 重命名为 `InvokeResult`，`RPCResult<T>` 重命名为 `InvokeResult<T>`
- `InvokeResult<T>` 独立 struct（C# struct 不支持继承），通过 `From(InvokeResult)` 工厂方法转换
- 移除 `AutoReconnect` / `ReconnectInterval` 属性，改为 `_userDisconnected` 标记控制重连
- 重连逻辑：手动 `Disconnect()` → 不重连；意外断线 → 自动重连（固定3秒间隔）；`Dispose()` → 不重连
- `ConnectAsync()` 重置 `_userDisconnected = false`，`Disconnect()` 设为 `true`
- `InvokeResult` 和 `InvokeResult<T>` 改为 class（非 struct），便于空检查和多处引用
- `ParseResponseMessages`（IEnumerable）改为 `ParseResponseMessage`（单条），一条数据行=一条响应
- 删除 `InvokeResult<T>` 和 `InvokeAsync<T>()`，只有 `InvokeAsync()` 返回 `InvokeResult`
- `InvokeResult` 新增 `GetValue<T>()` 方法，由调用方自行转换强类型值
- 完全移除 `InvokeResult` 类，`InvokeAsync()` 直接返回 `ResponseMessage`
- 错误/描述信息统一使用 `ResponseMessage.Description` 字段
- `PendingCall.SetResult` 直接透传 `ResponseMessage`，不再做额外包装
- `PendingCall.SetError` 创建仅含 `Code`/`Description`/`Timestamp` 的 `ResponseMessage`
- 新增 `CreateErrorResponse(int code, string description)` 私有静态辅助方法
- `CancelAllPendingCalls` 参数由 `error` 重命名为 `description`
- `InvokeAsync`/`NotifyAsync`/`InvokeInternalAsync` 的 `parameters` 从 `string` 改为 `object[]`
- null 处理：通过三元表达式选择 `Create(objectName, methodName)` 或 `Create(objectName, methodName, parameters)` 避免重载歧义
- `SerializeInvokeMessage` 后增加了 `bytes == null || bytes.Length == 0` 检查
